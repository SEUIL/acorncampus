
학습목표

1. 추상클래스와 인터페이스를  작성하는 방법을 알 수 있다
(미완성 설계도인 abstract와 규격/약속인 interface의 차이와 구현 방법 학습)
   - 추상클래스: 하나 이상의 추상 메서드를 포함하며, 상속을 통해 자식 클래스에서 완성하도록 유도하는 클래스 (is-a)
   - 인터페이스: 모든 메서드가 추상 메서드인 기본 설계도로, 클래스 간의 결합도를 낮추고 다중 상속을 지원 (has-a / 규격)

2. Object 클래스의 용도를 이해할 수 있다. - toString() , equals()  오버라이드  할 수있다.
(자바 최상위 클래스로서 객체의 정보를 출력하거나 논리적 동등성을 비교하는 핵심 메서드 재정의)
   - toString(): 객체의 문자열 표현을 반환 (기본은 주소값이나, 멤버 변수 값을 출력하도록 재정의)
   - equals(): 두 객체가 논리적으로 같은지 비교 (기본은 주소 비교이나, 데이터 값을 비교하도록 재정의)

3. Wrapper클래스가 무엇인지 알 수 있다( wrapper클래스의 용도를 알 수 있다)
(기본 자료형을 객체로 다루기 위해 포장하는 클래스로, 박싱과 언박싱의 개념 이해)
   - Boxing: 기본 타입(int) -> Wrapper 객체(Integer) / Unboxing: 그 반대 과정

4. String클래스의 내부 구조를 알 수 있다. ( char[]  + 매서드 )
(문자열이 실제로는 문자 배열로 저장되며 불변(immutable)의 특성을 가짐을 이해)
   - 불변성: 한 번 생성된 String 객체는 내용을 변경할 수 없으며, 변경 시 새로운 객체가 생성됨

5. String클래스의 주요기능들을 사용할 수 있다.
(substring, split, replace 등 문자열 조작을 위한 다양한 API 활용)

6. StringBuilder클래스를 사용할 수 있다.
(문자열의 잦은 변경이 일어날 때 메모리 효율을 높이기 위한 가변(mutable) 문자열 처리 학습)
   - 가변성: 내부 버퍼를 사용하여 새로운 객체 생성 없이 문자열을 추가(append)하거나 수정 가능

7. 업캐스팅과 다운캐스팅의 불편함을 해결해주는 제네릭을 클래스를 이해하고 사용할 수 있다.
(타입 안정성을 높이고 불필요한 형변환을 줄이는 Generic 문법 적용)
   - Generic: 클래스 내부에서 사용할 데이터 타입을 외부에서 지정하는 기법으로, 컴파일 시점에 타입 체크 수행

   ArrayList -기본버전, Generic버전 사용하기
   (Object 배열 기반의 리스트와 타입이 지정된 리스트의 차이 비교)

8. ArrayList 사용하여 간단한 성적처리하기
(동적 배열 구조를 활용하여 데이터를 저장, 관리하는 실무적인 로직 구현)




Object가 있는 이유

- 자바에서 만들어지는 모든 객체는 Object type으로 다룰 수 있다 (참조할 수 있다)
 이유는 모든 클래스는 Object를 상속받게 되어 있기 때문이다 ! 

- 라이브러리를 제공할 때 
 모든객체를  Object로 다룰 수 있게 되었다.

class Acorn{

        String id;
        String name;
        String phone;

       public Acorn(  String id, String name, String phone ) {
          this.id =id;
          this.name =name;
          this.phone =phone;
      }

      public void printInfo( ){
        System.out.println( id ) ;
        System.out.println( name );
        System.out.println( phone );
     }
}
